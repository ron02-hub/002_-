# クイックスタート - テストラン

## 🎯 今すぐテストランする方法

### 方法1: UIのみ確認（データベース不要・約1分）

```bash
cd src
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) にアクセス。

**確認できること**:
- ✅ ランディングページの表示
- ✅ UI/UXの確認

**制限**:
- ❌ アンケートを進めるとデータベースエラーが発生

### 方法2: 完全なテスト（データベース必要・約10分）

#### ステップ1: Supabaseでデータベース作成（5分）

1. https://supabase.com にアクセス
2. 「Start your project」→ GitHubでログイン
3. 「New Project」をクリック
4. プロジェクト名とパスワードを設定
5. リージョン: `Tokyo`
6. 「Create new project」をクリック
7. 2-3分待機

#### ステップ2: 接続情報を取得（1分）

1. 「Settings」→「Database」に移動
2. 「Connection string」→「URI」を選択
3. 接続文字列をコピー

#### ステップ3: 環境変数を設定（1分）

`src/.env.local`を開いて、以下を設定：

```env
DATABASE_URL="コピーした接続文字列"
NEXT_PUBLIC_APP_URL="http://localhost:3000"
NODE_ENV="development"
```

**重要**: 接続文字列の`[YOUR-PASSWORD]`を、プロジェクト作成時のパスワードに置き換えてください。

#### ステップ4: マイグレーション実行（2分）

```bash
cd src
npx prisma migrate dev --name init
npm run db:seed
```

#### ステップ5: サーバー起動（1分）

```bash
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) にアクセス。

## 🆘 困ったときは

- **データベース接続エラー**: `.env.local`の`DATABASE_URL`を確認
- **マイグレーションエラー**: `npx prisma migrate reset`でリセット
- **詳細は「テストラン手順.md」を参照**
